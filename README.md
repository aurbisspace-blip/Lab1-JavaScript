1.

var

Область видимости: функция (function scope), а не блок.

Можно переобъявлять: да.

Можно менять значение: да.

Hoisting: поднимается вверх и инициализируется undefined.

Часто ведёт к багам → сейчас почти не используют вижуально.
===================================================================
let

Область видимости: блок { } (if, for, while и т.д.).

Можно переобъявлять: ❌ нельзя в одном блоке.

Можно менять значение: ✅ можно.

Hoisting: есть, но без инициализации (до объявления использовать нельзя).
===============================================================================
const

Область видимости: блок { }.

Переобъявлять: ❌ нельзя.

Менять значение: ❌ нельзя (но объекты/массивы можно изменять внутри).

Hoisting: как у let.

======================================================================================
Hoisting (поднятие) в JavaScript — это механизм, при котором объявления переменных,
функций и классов интерпретируются как находящиеся в начале области видимости перед выполнением кода,
что позволяет использовать их до фактического объявления



2. Неявное преобразование типов

Это поведение JavaScript (его еще называют Type Coercion),
когда он автоматически меняет тип данных одного значения на другой, чтобы выполнить операцию.

Чаще всего это происходит при использовании операторов +, -, ==, и в условиях if.

Строковое преобразование: Если один из операндов при сложении — строка, второй тоже станет строкой.
'5' + 2 // "52" (Магия превращения числа в текст)

Числовое преобразование: Математические операторы (кроме +) превращают значения в числа.
'6' / '2' // 3
true + 1 // 2 (true превращается в 1)

Логическое преобразование: Происходит в условиях. Значения 0, "", null, undefined, NaN превращаются в false, остальное — в true.

3. Оператор == против ===
Разница в одном слове: строгость.

Нестрогое равенство (==)
Перед сравнением JavaScript пытается привести оба значения к одному типу. Это порождает знаменитые странности JS:

5 == "5" // true (строка стала числом)

null == undefined // true

0 == false // true

Строгое равенство (===)
Сравнивает и значение, и тип. Если типы разные — сразу false. Никаких скрытых превращений.

5 === "5" // false

0 === false // false
